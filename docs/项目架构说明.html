项目架构说明

当前项目主链路已经统一为 gRPC，历史 mprpc 模块和自研 fiber 协程模块已移除。

核心分层

  1. Clerk：客户端入口，负责向 KVServer 发起 Get/Put/Append 请求。
  2. KVServer：对外提供 gRPC KV 服务，并将写入请求提交给 Raft。
  3. Raft：负责选举、日志复制、提交和快照。
  4. raftRpcPro：维护 Raft 节点间与 KV 服务的 protobuf/gRPC 接口。
  5. skipList：作为 KV 状态机的底层存储结构。
  6. common：通用工具、配置读取、日志和阻塞队列。

通信路径

  • 客户端到 KVServer：kvServerRPC.proto 定义的 gRPC 服务。
  • Raft 节点之间：raftRPC.proto 定义的 gRPC 服务。
  • 节点配置：由 NodeConfig 读取 nodeXip/nodeXport 配置。

并发模型

Raft 的心跳、选举超时、apply loop 目前使用标准 C++ 后台线程运行。后续如果要继续现代化，可以在这个边界上引入线程池、任务执行器，最后再评估 C++20 coroutine。
